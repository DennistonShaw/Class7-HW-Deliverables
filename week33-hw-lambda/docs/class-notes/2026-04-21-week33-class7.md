> [!NOTE]
> ### Supplemental note  
> **Lambda Code Workflow: Zip vs Inline Editor**
>
> **Option 1 — Inline Editor (Simple / Lab Use)**
> - Write or paste code directly in AWS Lambda console
> - Click **Deploy**
> - Test immediately
>
> Use this when:
> - Code is small
> - No external files or libraries
> - Quick testing or learning
>
> **Option 2 — Zip Upload (Real Deployment)**
> - Write code in VS Code (or locally)
> - Save file(s) (e.g., `lambda_function.py`)
> - Zip the files into one `.zip`
> - Upload `.zip` to AWS Lambda
> - AWS automatically unzips and runs your code
>
> Use this when:
> - You have multiple files
> - You use external libraries
> - You are deploying with Terraform or automation
>
> **Key Difference**
> - Inline = fast and simple (paste → deploy)
> - Zip = structured and scalable (build → package → upload)

> [!NOTE]
> ### How API Gateway URLs Work
>
> API Gateway gives you a **base URL**, but that alone does not call your Lambda.
>
> You must append additional parts to reach a specific function.
>
> #### URL Structure
>
> ```
> Base URL + Stage + Route + Query Parameters
> ```
>
> #### Example
>
> ```
> https://abc123.execute-api.us-east-1.amazonaws.com/prod/node?name=Dennis
> ```
>
> #### Breakdown
>
> - Base URL → API Gateway endpoint  
> - Stage → `/prod` (deployed version of the API)  
> - Route → `/node` (determines which Lambda runs)  
> - Query Parameter → `?name=Dennis` (input passed to the function)
>
> #### Key Idea
>
> - The base URL alone returns **"Not Found"**  
> - The **route** tells API Gateway which Lambda to execute  
> - Query parameters become part of the **event input**

### Break Test — Missing Query String

Working URL:
`/prod/node?name=Dennis`

Broken URL:
`/prod/node`

Result:
- `rawQueryString` changed from `"name=Dennis"` to `""`
- `queryStringParameters` was missing
- Lambda used the fallback value: `"Unknown"`
- Response returned: `"HELLO UNKNOWN, DENNIS, THEO, AARON FROM NODE!"`
